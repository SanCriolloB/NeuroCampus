### 1.3 POST `/datos/validar`

Valida un archivo de datos **sin** almacenarlo. Ejecuta cadena: **esquema â†’ tipos â†’ dominio â†’ duplicados â†’ calidad**.

**Body (multipart/form-data)**

**200 â€” Response**


**CÃ³digos**
- `200` validado con Ã©xito
- `400` error al procesar archivo / formato invÃ¡lido

**Notas**
- El engine por defecto es **`pandas`**, configurable con `NC_DF_ENGINE=polars`.
- No persiste ni transforma el archivo; es solo validaciÃ³n para reporte en UI.
- El contrato de respuesta corresponde a `DatosValidarResponse` (Pydantic).

#### ðŸ†• NormalizaciÃ³n de encabezados y equivalencias
La validaciÃ³n **tolera nombres de columnas con espacios o con `_`**, acentos y signos como `:`.  
Internamente se normalizan (minÃºsculas, sin acentos, sin puntuaciÃ³n) y se aplican sinÃ³nimos bÃ¡sicos.

**Ejemplos de equivalencias admitidas:**
- `codigo_materia` â‰¡ `codigo materia` â‰¡ `CÃ³digo Materia` â‰¡ `codigo_asignatura`
- `cedula_profesor` â‰¡ `cedula profesor` â‰¡ `cÃ©dula docente` â‰¡ `docente_id`
- `Sugerencias:` â‰¡ `sugerencias` â‰¡ `Sugerencias`  
*(Se ignoran columnas `Unnamed: N` y se mantienen los nombres canÃ³nicos en el reporte).*

#### ðŸ†• CoerciÃ³n de tipos antes de validar
Si el esquema define el tipo de una columna, se intenta **coaccionar** el dato antes de reportar `BAD_TYPE`:
- `string`: convierte enteros/nÃºmeros a cadena (IDs como `codigo_materia`, `cedula_profesor`).
- `integer`: `to_numeric(...).astype("Int64")` (pandas) o `Int64` (polars).
- `number`: `float`.
- `boolean`: mapeo automÃ¡tico (`true/false/1/0`).
- `date/datetime`: `to_datetime(..., errors="coerce")`.
- **Nullables**: se respetan tipos del estilo `["string","null"]` (JSON Schema).

#### ðŸ†• Lectura de archivos (robusta)
- **CSV**: reintentos con `utf-8-sig` y `latin-1`; autodetecciÃ³n de separador (`; | tab | ,`) y manejo de BOM.
- **XLSX** con Polars: lectura vÃ­a **pandas** y conversiÃ³n a **polars**.
- **Parquet**: lectura nativa.

#### ðŸ†• Esquema aceptado
- **JSON Schema** (`properties`, `required`, `enum`, `minimum/maximum`) **o**
- Formato nativo: `{"columns":[{"name","dtype","domain{allowed|min|max}"}]}`.

#### ðŸ†• ResoluciÃ³n del schema en runtime
- El backend intenta localizar `schemas/plantilla_dataset.schema.json` subiendo directorios desde el cÃ³digo.
- Se puede forzar la ruta con `NC_SCHEMA_PATH=/ruta/absoluta/plantilla_dataset.schema.json`.
